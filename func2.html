Functions, Execution Context and the Call Stack

What does the computer do?


There are two mains thing the computer does when it runs a code:

  1. Store stuff in the memory
  2. Saves a "path" to each part of the memory that will be storing info.

Synchronous execution

When we go through the code line by line we go through it in order from top to bottom of our page and I never skip a line that means it is synchronous. Single threaded means one thing at a time. I am not starting to execute a function and continuing to do that while simultaneously continuing down the rest of a code at the same time. If I call a function my thread is not continuing down the code.

Functions

Functions are a way of wrapping up some instructions to be done later on. We give them a label. That saves us having to write all that code out again. So we just wrap it up and give it a label, and then we refer to it later.

function multiplyBy2(num){
    const result = num * 2;
    return result;
}

Remember the parenthesis. is the symbol that indicates there is some work to be done, so that the computar knows it should look for a function with the specific name and run any code stored in it. While executing lines inside the function, the variable to store the result is undefined.
